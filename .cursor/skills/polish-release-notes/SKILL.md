# ポーランド語のGenStudio リリースノート

**正規ターゲットファイル：** [help/user-guide/release-notes.md](../../../help/user-guide/release-notes.md)

**例：** [examples.md](examples.md)

**ワークフローの作成（アップストリーム）:** [generate-release-notes](../generate-release-notes/SKILL.md)

## 範囲（必須）

ユーザーが次のラウンドで&#x200B;**新たに追加した**&#x200B;と認識したコンテンツを&#x200B;**のみ** ポーランドします：

- **ファイル内：** `###`個のサブセクションが、`help/user-guide/release-notes.md`の単一の見出し`## YYYY.MM {#latest}`の下にあります。
- **以前のリリースノート** （折りたたみ可能な`+++` ブロック）の下にあるページタイトル、イントロ段落、フロントマター、またはテキストを&#x200B;**編集しないでください。**
- **ユーザー**&#x200B;明示的&#x200B;**が要求しない限り、同じ`{#latest}` セクション内の以前の`##`個の月間ブロックまたは既存の`###`を**&#x200B;編集しないでください。
- どの`###`が新しいかわからない場合は、**ask**&#x200B;か、**git diff** / edit contextを使用して解析を変更してください。
- **貼り付けられたコンテンツ：**&#x200B;は、ユーザーが&#x200B;**same**&#x200B;個の新しいサブセクションと一致することを確認したときにマークダウンを磨くだけです（無関係なペーストをインスコープとして扱わないでください）。

差分&#x200B;**最小限**&#x200B;を維持：文言と段落の区切りのみを保持します。ファイル内の他の場所ではドライブによるリファクタリングは行われません。

## 音声とトーン

- 製品リリースノートの&#x200B;**コピーライター**&#x200B;として書く：**簡潔**、**スキャン可能**、および&#x200B;**通電済み**&#x200B;約&#x200B;**新しい価値** – 成果を伴うリードと「何が役に立つのか」強力な動詞と鮮明なメリット言語を使用します。
- **正確さを維持：**&#x200B;機能が提供する内容を超える要求はありません。Experience LeagueとAdobeのトーンに一致します（自信に満ちた明確で、センセーショナリストではありません）。
- **特定の**&#x200B;のメリットと差分（マーケターが今すぐ実行できること、摩擦が取り除かれること）を持つ&#x200B;**興奮を生み出します。空の宣伝や根拠のない超過的要素は含まれません。**

## 段落ルール

- 各段落：**2 ～ 3個の文章**。 **1つの段落に3つ以上の文章**&#x200B;は使用できません。さらに必要な場合は、**新しい段落**&#x200B;を開始してください。
- オプション：レンダリングされたドキュメントの段落の長さが&#x200B;**～3行**&#x200B;を超える場合は、さらに分割します。

## 追加しない

- **手続き「方法」** コンテンツ：番号付きステップ、「クリック **[!UICONTROL ...]** then...」、完全なUI ウォークスルー、またはチュートリアルのフレーズ。 リリースノートでは、実践的なレッスンではなく、**何が出荷されたか**&#x200B;と&#x200B;**なぜそれが重要か**&#x200B;について要約します。
- 生成スキルで[禁止コンテンツ &#x200B;](../generate-release-notes/SKILL.md#prohibited-content)に違反するコンテンツ（Jira キー、内部専用URL、プルーフとしてのWikiなど）。

## 保存（構造的にストリップまたは書き換えしないでください）

- `[!DNL …]`、`[!UICONTROL …]`、`[!BADGE …]`およびその他のExL ショートコード。
- 既存の&#x200B;**相対** ドキュメントのリンクとアンカーパターン：`[phrase](/help/...)`を意味のあるアンカーテキストに配置します。
- Beta バッジは、[generate-release-notes examples](../generate-release-notes/examples.md)で使用されているとおりにブロックされます。

## ワークフローチェックリスト

1. [ ] `## … {#latest}`の&#x200B;**`###`がスコープに含まれていることを確認してください（このラウンドは新しいラウンドです）。**
2. [ ] スコープ内`###`ごとに、[音声とトーン &#x200B;](#voice-and-tone)および[段落ルール &#x200B;](#paragraph-rules)ごとにコピーを締めます。
3. [ ] **ハウツー**&#x200B;の指示を削除または短縮します。**ユーザーの結果**&#x200B;を保持します。
4. [ ] リンクとショートコードが引き続き有効であることを確認します。[品質チェック &#x200B;](#quality-checks)ごとに、内部IDまたは禁止されたパターンのクイックスキャンを実行します。

## 品質チェック

- [ ] `{#latest}`の下の合意された&#x200B;**new** `###` ブロックのみが変更されました。アーカイブと古い月は変更されません。
- [ ]新しいJira スタイル ID、内部Wiki URL、または「チケットを見る」言語はありません。
- [ ]個の段落は、それぞれ&#x200B;**2 ～ 3個の文章**&#x200B;です（1つの段落につき最大3個の文章）。
- [ ] コピーは&#x200B;**factual**&#x200B;のままであり、記述された機能と一致しています。
