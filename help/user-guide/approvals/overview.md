# Adobe GenStudio for Performance Marketingのレビューと承認

レビューと承認のワークフローにより、クリエイティブチームから法律専門家に至るまで、あらゆる関係者が、AI生成のブランドアセットを含むキャンペーンアセットとエクスペリエンスを効率的にレビューし、承認できるようになります。

>[!NOTE]
>
この機能は、Adobe Workfront Proof](/help/user-guide/approvals/proof-integration.md)との[統合としても利用できます。 この統合により、GenStudio for Performance Marketing Canvasでプルーフ機能が提供されます。 Workfront Proofとの統合により、GenStudio for Performance Marketingはより構造化され、透明性の高い、協力的なレビュープロセスを実現できます。これにより、チームはドラフトから最終的なレビューに至るまで、より確実かつ明確に進めることができます。

## レビューと承認ワークフローの利点

* **堅牢で反復的な生成AI コンテンツ作成のサポート**。 企業内でブランドに即したコンテンツを作成してデプロイすることは、非常に反復的なプロセスです。 GenStudio for Performance Marketingの生成AIを利用すれば、数百ものアセットのバリエーションをすばやく制作できます。 各審査担当者は、アセットを承認する前に、ドラフトに対して複数の変更を依頼できます。 レビュアーが多ければ多いほど、あらゆる関係者が最終的なバリエーションに同意する前に、潜在的な反復の数が多くなります。

* **クリエイティブの整合性のサポート**。 承認は、コンテンツ制作者を承認プロセスに関与させることにより、ブランドアセットのクリエイティビティを保護します。 レビューと承認のプロセスにクリエイティブ担当者（コンテンツ制作者やクリエイティブディレクターなど）を関与させることで、最終的な成果物がビジョンとブランドアイデンティティに沿ったものになります。

* **キャンペーン目標と法的要件の遵守**。 承認プロセスは、コンテンツがキャンペーン目標をサポートしているかどうかを検証するのに役立ちます。 これにより、あらゆるマーケティング資料が法律および規制基準に準拠していることを保証し、リスクと潜在的な法的問題を最小限に抑えることができます。

* **Adobe Workfront Proof**&#x200B;との統合。 また、GenStudio for Performance Marketing内から堅牢なレビューおよび承認機能を利用して、Workfront Proofにアクセスできます。 GenStudio for Performance MarketingでレビューされたコンテンツはWorkfront Proofに同期され、レビューコメントとステータスは保持されます。 [統合ハイライト ](/help/user-guide/approvals/proof-integration.md)は、ProofがGenStudio for Performance Marketingの承認ワークフローをどのように拡張するかを特定します。

## レビューと承認のライフサイクル

レビューと承認のワークフローの主なフェーズは次のとおりです。

* [作成したコンテンツのレビューと承認を依頼](/help/user-guide/approvals/request-review.md)。 GenStudio for Performance Marketingを使用すれば、承認のリクエストと承認者の管理のプロセスを効率化できます。 Workfront Proofの承認テンプレートを利用すれば、この作業をさらに簡素化できます。

* [ コンテンツのレビューと編集](/help/user-guide/approvals/review-and-edit.md)。 通知により、コンテンツ制作者は依頼された変更や承認について、常に最新情報を把握できます。 コンテンツのトリガーを見直し、新しい承認サイクルを自動化。

* [ コンテンツを承認](/help/user-guide/approvals/approve-content.md)。 指定された承認者が、コンテンツを「承認済み」または「公開可能」とマークします。

* [ コンテンツを公開](/help/user-guide/approvals/publish-content.md)。 承認済みコンテンツを[!DNL Content]に公開すると、組織内の他のユーザーが使用または参照できるようになります。

## コンテンツドラフトについて

_下書き_&#x200B;は、レビューと承認のプロセスを完了していないアセットまたはエクスペリエンスの予備バージョンです。 ドラフトのステータスは、ドラフトがレビューおよび承認プロセスのどの段階にあるかを示します。 一意のドラフト IDは、各ドラフトを識別します。 このIDは、ドラフトが承認され、[!DNL Content]に公開されるまで有効です。 レビューのコメントとドラフトの承認は、この個々のドラフト IDに関連付けられます。 GenStudio コンテンツドラフトのバージョン管理はありません。

ドラフトがレビューと承認のプロセスを完了し、[!DNL Content]に公開されると、ドラフト IDの有効期限が切れます。 GenStudio for Performance Marketingは、関連するコメントと承認ステータスを保存しません。 ドラフト URLは無効になりました。

ドラフトステータスは、レビューと承認のプロセスを進む際に、コンテンツドラフトの状態をキャプチャします。 レビュー中のアセットを作成したGenStudio for Performance Marketing コンテンツエディターには、ドラフトまたは承認に対する変更がリクエストされたことが通知されます。 承認者は、ドラフトのステータスを変更して、ドラフトをさらに修正する必要があるか、承認できるかどうかを示します。 あらゆる承認者は、アセットやエクスペリエンスを公開する前に承認する必要があります。

使用可能なドラフトのステータス：

**通知**: コンテンツ エディターは、ドラフトのレビューの準備が整ったことを承認者に通知することにより、レビューと承認のプロセスを開始しました。
**作業が必要**: 1人以上の承認者がコンテンツのドラフトの変更を要求したことを示します。 このステータスのコンテンツは[!DNL Content]に保存できません。
**承認済み**：指定されたすべての承認者がアセットまたはエクスペリエンスを承認しました。 コンテンツエディターがアセットまたはエクスペリエンスにメタデータを追加し、[!DNL Content]に保存できるようになりました。

>[!NOTE]
>
ドラフトは、Workfront Proof統合のユーザーの&#x200B;_プルーフ_&#x200B;に対応します。 [ ドラフトとプルーフ ](/help/user-guide/approvals/proof-integration.md#drafts-and-proofs)は、永続性とバージョン管理が異なります。

## 承認ロール

_レビュー担当者_&#x200B;はコメントを追加できますが、コンテンツを承認できません。 レビュアーの参加は役に立ちますが、不可欠ではありません。 _承認者_&#x200B;は、承認プロセスを進める前にコンテンツを承認する必要があります。 Workfront Proofとの連携により、より幅広い利用者の役割をサポートできます。

## 通知

GenStudio for Performance Marketingの製品内通知により、アセットのステータス変更や`@mention`件のコメントなど、承認者やコンテンツエディターはリアルタイムで更新されます。 通知は、複数のレビュー、編集、承認サイクルを通じて、迅速な反復をサポートします。

コンテンツエディターと承認者は、Slackでこれらの通知を受け取るために登録できます。 Experience Cloudでのサービスの購読[を参照してください](https://experienceleague.adobe.com/en/docs/core-services/interface/services/customer-attributes/subscription)。

承認参加者が実行したアクションは、製品内の通知やメール通知を自動的にトリガーします。 承認プロセスを開始すると、指定された承認者には、電子メールと製品内の両方の通知が届きます。 承認者が`@mention`件のコメントを追加したり、決定を下したりすると、常に製品内および電子メールで通知を受け取ることができます。 通知には、コンテンツのドラフトへのリンクが含まれます。

コンテンツのレビューと承認のプロセスを開始すると、すべての承認とレビューコメントが通知されます。 ただし、承認者には、`@mention`を含むコメントのみが通知されます。
